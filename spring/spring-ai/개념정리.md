### 🎯 주요 키워드

✔ LLM(Large Language Model)

- 거대한 언어 모델, 느리고 비싼 하나의 외부 API 서비스라 생각하면 될 것 같다.

-> CHAT GPT, Gemini, Cluad와 같은 '자연어를 다루는 대규모 언어 모델' 을 일컫는 말이다.

✔ Prompt

- LLM에게 전달하는 입력 명세서, LLM 모델에게 던지는 하나의 요구사항 정의서

```
INPUT : 안녕
TARGET : 미국인
OUTPUT : 영어 번역 텍스트
```

✔ Token
- LLM의 비용 및 처리 단위

-> API 호출 시, 생성되는 Payload와 Response의 

✔ Context Window

- LLM이 한 번에 기억할 수 있는 최대 토큰 수

-> 요청의 메모리 제한

✔ Hallucination
- 잘못된 응답
-> GPT에게 모르겠습니다를 듣기 어렵지만, 그만큼 이런 Hallucination이 섞여 있다

### 🎯 Spring AI란?

AI/LLM 기능을 자연스럽게 애플리케이ㅣ션에 결합할 수 있게 해주는 AI-엔지니어링 프레임워크

-> 결국 LLM과 Spring을 매끄럽게 연결해주는 프레임워크이다.


### 🎯 Spring AI 특징

1) 멀티 AI 모델 지원

다양한 LLM 서비스와 모델을 하나의 공통 API로 호출 할 수 있다.

✔ OpenAI, Anthropic, Microsoft Azure OpenAI
✔ Google Vertex AI (Gemini 포함)
✔ Amazon Bedrock
✔ Ollama 등 로컬/클라우드 모델

```java
ChatClient chatClient = builder.build();
String response = chatClient.prompt("Hello World!").call().content();
```

-> chatClient는 내부적으로 OpenAi, Gemini 등 여러 모델을 처리할 수 있도록 구현되어 있으며, 설정 변경만으로 모델 교체가 가능

2) Spring 방식 그대로의 개발
기존 Spring 개발처럼 진행 가능하다.

- @Configuration, @Bean, applicayion.yml
- DI 구조

-> 거대한 진입장벽 없이, 기존 Spring의 생태계가 조금 확장된것이다.

3) Prompt의 객체화

``` java
PromptTemplate template = new PromptTemplate("""
너는 {targetLanguage} 번역기다.
다음 문장을 번역해라:

{text}
""");

Prompt prompt = template.create(Map.of(
    "targetLanguage", "한국어",
    "text", "Hello, how are you?"
));

String response = chatClient
    .prompt(prompt)
    .call()
    .content();
```

-> 하드코딩 불필요, Prompt를 문자열이 아닌, 역할을 가진 설계된 객체로 관리할 수 있도록 지원한다.
